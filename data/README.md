# Data Directory

This directory is organized for reproducible acquisition and curation.

- `manifests/`: dataset metadata, split IDs, license notes, and default smoke-test limits.
- `raw/`: small downloaded samples from public datasets.
- `external_repos/`: cloned reference repositories such as `torchgfn`, `sfm`, and `unigenx`.
- `external_models/`: optional model-repo metadata and size-limited downloads.
- `processed/`: graphified JSONL used by training and validation.
- `samples/`: tiny hand-authored or generated examples for tests and demos.

Large corpora from the paper are not downloaded by default. Use the manifest and `scripts/acquire_datasets.py` with explicit dataset names and limits.

- `external_models/sfm/{FILES,SUMMARY}.json`
- `external_models/unigenx/{FILES,SUMMARY}.json`

- `processed/local_pubchem/train.jsonl`
- `processed/local_uniprot/train.jsonl`
- `processed/local_refseq/train.jsonl`

- `processed/local_pdbbind/train.jsonl`

These are generated by `scripts/prepare_science_sources.py` from local CSV, TSV, JSON, JSONL, or FASTA files. They are not downloaded automatically.

- `raw/hebrew_qa_nnlp/repo`

The Verb Complements Lexicon is not auto-downloaded. Place a TSV/CSV under `raw/hebrew_verb_complements_lexicon/` if a reviewed source is available; the preparation script will graphify supported `verb_root` rows.
